# Cobra NATS — Agent Configuration

## <agent_rules>
### UI Components (shadcn/ui)
- NEVER create UI components manually in `src/components/ui/`.
- ALWAYS use the shadcn CLI to add new components: `npx shadcn@latest add <component_name>`.
- Use the provided workflow `/add-component` when adding components.

### Server Actions
- ALL new NATS operations must be Next.js Server Actions placed in `src/app/actions/`.
- ALWAYS use the wrappers `withNatsConnection()` or `withJetStream()` from `action-helpers.ts`.
- EVERY action must return the `ActionResponse<T>` format.
- Use the provided workflow `/create-server-action` when creating new actions.

### Testing
- Functional tests (`functional-*.spec.ts`) require a running NATS server.
- ALWAYS ensure `docker-compose up -d` is running before executing functional tests.
- When running tests via Playwright, default to the Chromium project if not specified otherwise (`--project=chromium`).
## </agent_rules>

## Project Overview

Manages streams, KV stores, message publishing, and live traffic monitoring.

## Architecture

### Server Actions Pattern

All NATS operations run as **Next.js Server Actions** in `src/app/actions/`.
Credentials never leave the server. Every action returns `ActionResponse<T>` which is a strict Discriminated Union:

```typescript
export type ActionResponse<T = void> =
  | { success: true; data: T }
  | { success: false; error: string };
```

Helper wrappers: `withNatsConnection()` and `withJetStream()` in `action-helpers.ts`.
ALWAYS check `result.success` to narrow the union type before accessing `result.data`. NEVER cast errors or results to `any`.

### Connection Management

- `NatsManager` (singleton) manages NATS connections, JetStream managers/clients.
- `useNatsStore` (Zustand) persists connection configs in `localStorage`.
- Types: `NatsConnectionConfig`, `NatsConfig` in `src/lib/nats/nats-types.ts`.

### SSE Endpoint

Live monitoring uses Server-Sent Events via `src/app/api/monitor/route.ts`.

## Conventions

- **@no-explicit-any**: NEVER use `any` types. ALL variables, parameters, and returns must be strictly typed.
- **Error Handling**: Use `err: unknown` in `catch` blocks and format with `getErrorMessage(err)`.
- **Testing**: ALWAYS verify changes against the Playwright tests (`npx playwright test`).
- **Connection Config**: Use the unified `NatsConnectionConfig` type for all frontend-to-backend connection passing.
- **Code comments** in English.
- **Imports** use `@/` path alias (maps to `./src/`).
- **UI theme**: dark-mode only (`html.dark`), Tailwind slate palette.
- **Components in `ui/`** are generated by shadcn — do not edit manually.
- **New pages** go under `src/app/(dashboard)/`.
- **New server actions** go in `src/app/actions/`, use `withJetStream()` or `withNatsConnection()`.
- **New NATS types** go in `src/lib/nats/nats-types.ts`.
